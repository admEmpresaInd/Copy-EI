02 — Claude Code
🤖

O que é Claude Code?

Uma IA que programa com você usando linguagem natural, direto no terminal.

Claude Code é um agente de IA desenvolvido pela Anthropic que roda no seu terminal. Em vez de abrir o ChatGPT, copiar código e colar em algum lugar, o Claude Code age diretamente — ele lê seus arquivos, escreve código, cria projetos inteiros, faz commits no Git e muito mais, tudo a partir de uma conversa em texto.

💬 Linguagem natural

Você fala o que quer em português. O Claude entende o contexto e escreve o código correto.

📁 Acessa seus arquivos

Ele lê e edita arquivos do seu projeto em tempo real — não é só sugestão, é ação real.

🔁 Conversa contínua

Mantém contexto da conversa. Você pode pedir ajustes, correções e refinamentos iterativamente.

🛠️ Executa comandos

Roda npm install, git commit, testa código — tudo sem você precisar digitar cada comando.

💡

Analogia: imagine ter um programador sênior sentado ao seu lado, disponível 24h. Você descreve o que precisa em português, ele escreve, você aprova ou pede ajuste. É exatamente isso que o Claude Code faz — só que no terminal do seu computador.

03 — Skills
⚡

O que são Skills?

Módulos especializados que ampliam as capacidades do Claude Code.

Skills são como aplicativos instalados no Claude Code. Elas carregam instruções, frameworks, ferramentas e contexto especializado para áreas específicas: copywriting, design de interfaces, análise de mercado, deploy, e muito mais. Quando você digita /nome-da-skill, o Claude muda de modo e passa a operar com expertise naquele domínio.

💡

Analogia: pense nas Skills como consultores especialistas que você pode acionar. O Claude base é um generalista excelente. Quando você ativa a skill /copy-senior, ele vira um copywriter especialista em direct response. Quando ativa /frontend-design, vira um designer de interfaces premium.

// skills disponíveis no seu programa

/copy-senior Copywriter sênior de direct response — VSLs, ads, páginas de venda, emails Copy & Persuasão
/frontend-design Cria interfaces web e landing pages com design premium e produção real Design & UI
/marketing-skills 42 skills de marketing: SEO, CRO, conteúdo, campanhas, growth, inteligência Marketing
/instagram-post Cria e publica conteúdo no Instagram — estático ou carrossel Social Media
/engineering-skills 23 skills de engenharia: arquitetura, fullstack, DevOps, segurança, IA/ML Engenharia
/alex-hormozi-pitch Cria ofertas irresistíveis usando o framework de $100M Offers Vendas & Oferta
ℹ️ Skills ficam na pasta .claude/skills/ do seu projeto. Você pode criar skills próprias, compartilhá-las com a equipe e instalar novas a qualquer momento.
04 — GitHub
🗂️

O que é GitHub?

A plataforma onde o código do seu projeto vive, com histórico e colaboração.

GitHub é uma plataforma baseada no Git, um sistema de controle de versão. Ele guarda todo o histórico do seu código — cada alteração, quem fez, quando. Funciona como um Google Drive inteligente para código, mas muito mais poderoso: permite colaboração, rollback de mudanças, branches para testar features, e integração automática com Vercel para deploy.

📦 Repositório

Uma pasta no GitHub que contém todo o código de um projeto. Pública ou privada.

💾 Commit

"Salvar com mensagem" — cada commit registra o que mudou e por quê. É o histórico do projeto.

🌿 Branch

Uma versão paralela do código para testar algo sem afetar o principal (main).

🔀 Push / Pull

Push = enviar alterações para o GitHub. Pull = baixar alterações do GitHub.

💡

Analogia: o GitHub é como um Google Drive com máquina do tempo. Você salva seu código, pode ver como estava em qualquer data no passado, restaurar versões antigas, ter múltiplas "cópias paralelas" do projeto e trabalhar com outras pessoas sem conflito.

// sem controle de versão

❌ site_final_v2_DEFINITIVO_2.html
❌ site_copia_backup_novo.html
❌ "Qual era a versão que funcionava?"
❌ Medo de mudar e quebrar tudo

// com GitHub

✅ Um arquivo só, histórico completo
✅ git log → ver todas as mudanças
✅ git checkout → voltar a qualquer versão
✅ Edite com confiança, sempre tem backup

05 — Supabase
🗄️

O que é Supabase?

O banco de dados, autenticação e storage dos seus projetos — tudo em um lugar.

Supabase é um backend como serviço (BaaS) open-source. Ele oferece banco de dados PostgreSQL, sistema de login/autenticação, armazenamento de arquivos e APIs prontas — tudo sem precisar configurar um servidor. É o que permite que sua aplicação salve dados, tenha usuários logados e guarde imagens.

🗃️ Banco de Dados

PostgreSQL completo. Tabelas, relacionamentos, queries SQL. Você cria a estrutura no painel visual.

🔐 Autenticação

Login com email, Google, GitHub etc. Gerenciamento de usuários e sessões já prontos.

📸 Storage

Armazenamento de arquivos (imagens, PDFs, vídeos) com URLs públicas ou privadas.

⚡ Realtime

Atualizações em tempo real — quando alguém salva no banco, outros usuários veem na hora.

💡

Analogia: pense no Supabase como o "Google Firebase mas melhor". É o lugar onde sua aplicação guarda informações — os cadastros de clientes, os pedidos, as fotos de perfil, as sessões de login. Sem ele, sua aplicação não tem memória.

✅ O plano gratuito do Supabase inclui 500MB de banco de dados, 1GB de storage e 50.000 usuários — mais do que suficiente para começar qualquer projeto.
06 — Vercel
🚀

O que é Vercel?

A plataforma que coloca sua aplicação na internet em segundos, automaticamente.

Vercel é uma plataforma de hospedagem e deploy contínuo. Você conecta seu repositório do GitHub e a Vercel faz o resto: toda vez que você faz um push para o GitHub, a Vercel automaticamente atualiza o site ao vivo. Você nunca precisa fazer upload de arquivos manualmente, configurar servidores ou se preocupar com infraestrutura.

🔄 Deploy automático

Push no GitHub → site atualizado em 30 segundos. Automático, sem nenhuma ação extra.

🌐 URL instantânea

Cada projeto ganha uma URL pública imediatamente (ex: meu-projeto.vercel.app).

🔒 HTTPS grátis

SSL/HTTPS configurado automaticamente. Seu site é seguro sem custo extra.

📊 Analytics

Vê quantas pessoas acessaram, de onde, qual performance — painel completo.

💡

Analogia: Vercel é o "Shopify da hospedagem de aplicações". Em vez de contratar servidor, configurar DNS, instalar certificados de segurança... você conecta seu GitHub, e em 2 cliques seu projeto está online, com URL, HTTPS e deploy automático.

1

Instalar o Homebrew

Verificar / Instalar o Winget

O gerenciador de pacotes do Mac — facilita instalar qualquer ferramenta de desenvolvimento.

O gerenciador de pacotes do Windows 11 — já vem instalado na maioria dos sistemas modernos.

💡

O que é o Homebrew? É como a App Store, mas para ferramentas de desenvolvedor. Com um comando você instala qualquer programa sem precisar procurar no Google, baixar instalador, arrastar para Applications...

Cole este comando no Terminal e pressione Enter:

Terminal (macOS) 
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
⚠️ O processo vai pedir sua senha do Mac. Você não vai ver os caracteres enquanto digita — isso é normal. Digite e pressione Enter.

Se você usa Mac com chip Apple Silicon (M1/M2/M3/M4), após a instalação o terminal vai mostrar instruções para adicionar o Homebrew ao PATH. Execute os comandos sugeridos (começam com echo e source).

Verifique se instalou corretamente:

Verificação 
$ brew --version
Homebrew 4.x.x   ← deve aparecer algo assim ✅
⚠️ Abra o PowerShell como Administrador para TODOS os passos desta instalação: pressione Win + X e escolha "Terminal (Admin)" ou "Windows PowerShell (Admin)". Mantenha ele aberto durante toda a instalação.
💡

O que é o winget? É o gerenciador de pacotes oficial da Microsoft, embutido no Windows 11. Funciona como o Homebrew do Mac: um comando instala qualquer ferramenta sem baixar instaladores manualmente.

Verifique se o winget está disponível:

PowerShell (Admin) 
PS> winget --version
v1.x.x   ← deve aparecer assim ✅
ℹ️ Windows 11: winget já vem instalado. Windows 10: instale o "App Installer" na Microsoft Store e reinicie o PowerShell.

Libere a execução de scripts no PowerShell (necessário apenas uma vez):

PowerShell (Admin) 
PS> Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
Execution policy changed. ✅
✅ Este comando permite ao PowerShell executar scripts e pacotes instalados via npm. Precisa ser feito apenas uma vez.

Instalar o Node.js

O ambiente de execução JavaScript que o Claude Code precisa para funcionar.

Vamos usar o nvm (Node Version Manager) — permite ter múltiplas versões do Node e trocar entre elas facilmente.

// passo 2.1 — instalar o nvm

Terminal (macOS) 
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

Depois de instalar, feche e reabra o Terminal, e então instale o Node.js:

// passo 2.2 — instalar Node.js LTS (versão estável)

Terminal (novo) 
$ nvm install --lts
Installing latest LTS version...
Now using node v22.x.x

$ node --version
v22.x.x   ← deve aparecer assim ✅

$ npm --version
10.x.x   ← npm instalado junto ✅

Vamos usar o nvm-windows (Node Version Manager para Windows) — instalado via winget.

// passo 2.1 — instalar o nvm-windows

PowerShell (Admin) 
PS> winget install CoreyButler.NVMforWindows
Successfully installed. ✅

Feche e reabra o PowerShell como Admin, e então instale o Node.js:

// passo 2.2 — instalar Node.js LTS

PowerShell (Admin — novo) 
PS> nvm install lts
Downloading and installing Node.js LTS...

PS> nvm use lts
Now using node v22.x.x

PS> node --version
v22.x.x   ← deve aparecer assim ✅

PS> npm --version
10.x.x   ← npm instalado junto ✅
✅ No nvm-windows, sempre use nvm use lts após reiniciar o PowerShell para ativar a versão correta do Node.

Instalar o Cursor

O editor de código com IA integrada — onde você vai visualizar e trabalhar nos seus projetos.

💡

O que é o Cursor? É um editor de código baseado no VS Code, com IA integrada. Você escreve, edita e visualiza seus arquivos aqui. O Claude Code roda no terminal integrado do Cursor — os dois trabalham juntos no mesmo ambiente.

Instale via Homebrew:

Terminal (macOS) 
$ brew install --cask cursor
cursor was successfully installed! ✅
ℹ️ Alternativa: baixe o instalador diretamente em cursor.com e arraste para a pasta Applications.

Instale via winget:

PowerShell (Admin) 
PS> winget install Anysphere.Cursor
Successfully installed. ✅
ℹ️ Alternativa: baixe o instalador diretamente em cursor.com e execute o arquivo .exe.

// configuração inicial

  • Abra o Cursor — aparece no Launchpad (Mac) ou no Menu Iniciar (Windows)
  • Clique em "Sign In" e crie uma conta gratuita em cursor.com
  • Na tela de configuração, pode pular as opções de IA por enquanto — vamos usar o Claude Code no terminal
  • Familiarize-se com a interface: é idêntica ao VS Code — mesmos atalhos, mesmas extensões

⌘K Gerar código inline

Abre o prompt dentro do arquivo para pedir ao Cursor que escreva ou edite código na posição do cursor.

⌘L Chat lateral

Chat com contexto do projeto inteiro — ótimo para perguntas, análise e refatoração de código.

⌃` Terminal integrado

Abre o terminal dentro do Cursor — é aqui que você vai rodar o Claude Code nos próximos passos.

✅ No Windows os atalhos usam Ctrl no lugar de ⌘: Ctrl+K, Ctrl+L e Ctrl+`.

user.email=seu@email.com   ← ✅

// 4.3 — criar chave SSH (conexão segura com o GitHub)

A chave SSH é como uma senha permanente e automática que identifica seu computador para o GitHub.

Gerar chave SSH (macOS) 
# gerar a chave (substitua pelo seu email)
$ ssh-keygen -t ed25519 -C "seu@email.com"

Generating public/private ed25519 key pair.
Enter file in which to save the key: ← pressione Enter
Enter passphrase: ← pressione Enter (sem senha)
Your identification has been saved in ~/.ssh/id_ed25519 ✅

# adicionar ao ssh-agent
$ eval "$(ssh-agent -s)"
$ ssh-add ~/.ssh/id_ed25519

# copiar a chave pública
$ cat ~/.ssh/id_ed25519.pub
ssh-ed25519 AAAAC3Nza... seu@email.com   ← copie TUDO isso
Gerar chave SSH (Windows — PowerShell) 
# gerar a chave (substitua pelo seu email)
PS> ssh-keygen -t ed25519 -C "seu@email.com"

Generating public/private ed25519 key pair.
Enter file in which to save the key: ← pressione Enter
Enter passphrase: ← pressione Enter (sem senha)
Your identification has been saved in C:\Users\Voce\.ssh\id_ed25519 ✅

# adicionar ao ssh-agent (habilite o serviço primeiro)
PS> Get-Service ssh-agent | Set-Service -StartupType Automatic
PS> Start-Service ssh-agent
PS> ssh-add $env:USERPROFILE\.ssh\id_ed25519

# copiar a chave pública (PowerShell)
PS> Get-Content $env:USERPROFILE\.ssh\id_ed25519.pub
ssh-ed25519 AAAAC3Nza... seu@email.com   ← copie TUDO isso

// 4.4 — adicionar chave no GitHub

  • Acesse github.com → Settings → SSH and GPG keys
  • Clique em "New SSH key"
  • No campo Title, escreva "Meu Mac"
  • No campo Key, cole tudo que copiou no passo anterior
  • Clique em "Add SSH key" e confirme com sua senha do GitHub

// 4.5 — testar conexão

Teste SSH 
$ ssh -T git@github.com
Hi seu-usuario! You've successfully authenticated ✅

// 4.6 — instalar GitHub CLI (opcional mas útil)

GitHub CLI (macOS)

$ gh auth login   # siga as instruções para autenticar
GitHub CLI (Windows) 
PS> winget install GitHub.cli
# feche e reabra o PowerShell, depois:
PS> gh auth login   # siga as instruções para autenticar
